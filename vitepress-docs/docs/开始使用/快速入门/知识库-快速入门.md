# 知识库快速入门

知识库是基于 **RAG（检索增强生成，Retrieval-Augmented Generation）** 技术构建的知识管理模块。通过将私域文档向量化存储，在用户提问时自动检索相关内容并注入上下文，从而使大模型能够基于真实文档进行精准回答，有效降低模型幻觉（Hallucination）。

**核心能力：**
- 支持多格式文档上传（PDF、Word、TXT、Markdown 等）
- 文档自动分块与向量化处理
- 语义相似度检索 + Rerank 精排
- 多知识库多路召回融合
- 引用溯源与原文下载

---

## 第一步：关联知识库

在使用知识库之前，需要先将知识库与助手应用进行关联。

1. 进入目标助手应用的**参数配置**页面
2. 找到「知识库」配置区域，点击 **添加知识库** 按钮
3. 在弹出的知识库列表中，勾选一个或多个需要关联的私域知识库，点击确认

![关联知识库](/assets/知识库/知识库使用1.png)

### 多知识库召回配置

当关联了**多个知识库**时，建议开启多路召回并配置 Rerank 模型，以提升检索质量：

| 配置项 | 说明 | 推荐值 |
|--------|------|--------|
| **召回模式** | 选择「多路召回」，同时从多个知识库并行检索 | 多路召回 |
| **Rerank 模型** | 对多路召回结果进行精排，过滤低相关内容 | 按需选择 |
| **Top K** | 最终注入上下文的文档片段数量 | 3 ~ 6 |
| **Score 阈值** | 相似度得分低于该值的片段将被过滤，范围 0 ~ 1 | 0.5 ~ 0.7 |

> **提示：** Top K 值越大，召回内容越丰富，但也会增加 Token 消耗；Score 阈值越高，结果越精准，但可能导致召回率下降。建议根据实际业务场景调整。

## 第二步：开启引用与归属

为了让回答结果可追溯，需要在会话配置中开启引用显示功能。

1. 进入助手应用的**会话增强**配置区域
2. 开启 **引用** 开关 —— 回答中将展示所引用的知识库文档片段
3. 开启 **归属** 开关 —— 回答中将标注内容来源的文档名称

配置完成后，在对话输入框中输入与知识库文档相关的问题，助手将自动检索并引用相关内容进行回答。

![引用知识库内容](/assets/知识库/知识库使用2.png)

## 第三步：查看与下载引用文档

当助手回答引用了知识库中的文档内容时，回复结果下方会展示**引用来源列表**。

**下载条件（需同时满足）：**
- 本次回答确实引用了知识库文档内容
- 当前用户账号具备文档下载权限

**操作步骤：**

1. 在回复结果下方找到引用来源区域
2. 点击对应文档右侧的 **下载图标**
3. 系统将下载该引用文档的原始文件

![下载引用文档](/assets/知识库/知识库使用3.png)

> **注意：** 若下载图标不可点击，请联系管理员确认您的账号是否已开通文档下载权限。

## 常见问题

**Q：为什么助手没有引用知识库内容？**

可能原因：
- 问题与知识库文档相关性较低，未达到 Score 阈值
- 知识库文档尚未完成向量化处理（可在知识库管理页查看索引状态）
- 知识库未正确关联到当前助手应用

**Q：如何提升检索准确率？**

- 适当降低 Score 阈值，扩大召回范围
- 增大 Top K 值，引入更多候选片段
- 优化文档分块策略，避免单块内容过长或过短
- 启用 Rerank 模型对召回结果进行二次精排
