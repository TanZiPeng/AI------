# 对话型业务流

对话型业务流编辑器用于编排基于多轮对话的 AI 应用流程。完成流程编排后，可通过页面右上角的工具栏进行检查、预览和发布操作。

![对话型业务流基础界面](/assets/调试发布/对话业务流-基础界面.png)

---

## 页面布局

| 区域 | 说明 |
|------|------|
| 左侧节点面板 | 提供可拖拽的节点类型，分为**节点**和**组件**两个标签页 |
| 中央画布 | 流程编排区域，通过连线将各节点串联成完整业务流 |
| 右上角工具栏 | 提供**检查清单**、**预览**、**功能**、**发布**四个操作入口 |

### 可用节点类型

左侧面板按功能分组列出所有可用节点：

| 分组 | 节点 |
|------|------|
| （基础） | 模型、知识库、临时知识库、直接回复 |
| 问题理解 | 意图识别 |
| 逻辑 | 条件选择、任务循环 |
| 转换 | 代码运行、整合输出、文本提取、文档拆分、参数提炼、变量赋值、对话SQL |
| 工具 | HTTP请求 |

---

## 预览调试

点击右上角**预览**按钮，页面右侧弹出预览面板，可在不发布的情况下实时测试业务流效果。

![预览-节点流转日志](/assets/调试发布/对话业务流-预览-节点流转日志.png)

在底部输入框中输入问题后发送，预览面板将同步展示：

- **对话回复**：AI 根据业务流配置生成的回答
- **业务流节点流转日志**：展示每个节点的执行状态、耗时及数据处理详情

### 节点日志说明

展开某个节点（如**模型**节点）可查看该节点的详细运行信息：

| 信息项 | 说明 |
|--------|------|
| 数据处理 | 节点接收到的输入数据，以 JSON 格式展示，包含消息历史、模型供应商、模型名称等 |
| 输出 | 节点执行后的输出结果，包含模型回复文本及 Token 用量统计（prompt_tokens、completion_tokens 等） |
| 耗时 | 该节点的执行耗时，显示在节点名称右侧（如 `2.755s`） |

> **提示：** 节点流转日志有助于排查业务流中的异常节点，建议在发布前充分测试各类输入场景。

## 发布与更新

预览验证通过后，点击右上角**发布**按钮展开发布菜单，提供以下两个操作：

![发布更新](/assets/调试发布/对话型业务流-发布更新.png)

| 操作 | 说明 |
|------|------|
| 发布更新 | 将当前业务流的最新配置发布为线上版本，已上线的应用将立即生效 |
| 运行 | 以当前已发布版本启动运行，跳转至应用运行界面 |

---

## 运行界面

点击**运行**后进入应用的对话运行界面，用户可在此与已发布的 AI 应用进行正式对话。

![运行界面](/assets/调试发布/对话型业务流-运行界面.png)

| 区域 | 说明 |
|------|------|
| 左侧历史会话 | 展示历史对话记录，点击**新建对话**可开启新会话 |
| 顶部应用名称 | 显示当前运行的业务流名称 |
| 对话设置 | 可展开配置对话相关参数 |
| 中央对话区 | 展示用户消息与 AI 回复的完整对话内容 |
| 底部输入框 | 输入问题或需求后点击发送按钮提交 |

> **注意：** 运行界面展示的是已发布版本的效果。若需测试未发布的修改，请使用**预览**功能。
