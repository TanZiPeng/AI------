# 创建组件

平台支持自定义创建组件，通过导入 OpenAPI Schema 定义接口，即可将任意 HTTP 服务封装为可在工作流中调用的组件。

进入**组件中心 → 我的组件**，点击左上角**创建我的组件**按钮，右侧弹出创建面板。

![创建我的组件](/assets/组件中心/组件中心-创建我的组件.png)

---

## 配置说明

| 配置项 | 是否必填 | 说明 |
|--------|----------|------|
| 名称 | 必填 | 组件的显示名称，建议简洁易识别 |
| Schema | 必填 | 符合 OpenAPI-Swagger 规范的接口描述，用于解析出可用组件列表 |
| 鉴权方法 | 选填 | 调用该组件时使用的鉴权方式，默认为**无**，可按需配置 |
| 标签 | 选填 | 为组件添加分类标签，便于管理和检索 |
| 隐私协议 | 选填 | 填写该组件对应服务的隐私协议链接或说明 |
| 自定义免责声明 | 选填 | 填写使用该组件时需展示的免责声明内容 |

---

## 填写 Schema

Schema 字段需填写符合 **OpenAPI-Swagger 规范**的 JSON 或 YAML 内容，系统会自动解析出**可用组件**列表，展示每个接口的名称、描述、请求方法和路径。

支持两种导入方式：

- **手动填写**：直接在输入框中粘贴 OpenAPI Schema 内容
- **从 URL 导入**：点击**从 URL 中导入**，输入 Schema 文件的在线地址，系统自动拉取并填充

> **提示：** 如不熟悉 OpenAPI 规范，可点击 Schema 旁的**查看 OpenAPI-Swagger 规范**链接查阅官方文档，或参考右侧**示例**快速了解格式。

---

## 鉴权方法

鉴权方法用于配置调用该组件接口时的身份验证方式，常见方式包括：

| 鉴权方式 | 说明 |
|----------|------|
| 无 | 接口无需鉴权，直接调用 |
| API Key | 在请求头或参数中携带密钥 |
| Bearer Token | 在请求头中携带 `Authorization: Bearer <token>` |
| Basic Auth | 使用用户名和密码进行基础认证 |

---

## 保存与使用

配置完成后点击**保存**，组件将出现在**我的组件**列表中。创建成功的组件可在工作流节点中直接引用调用。

> **注意：** Schema 解析后若**可用组件**列表为空，请检查 Schema 格式是否正确，或确认接口路径定义是否完整。
