# 部门管理

部门管理用于维护系统中的组织架构信息。管理员可通过顶部导航栏进入**用户中心 → 部门**页面，对部门进行查看、新增、编辑和删除操作，同时支持在已有部门下添加子部门，构建多层级的组织结构。

---

## 部门列表

部门列表展示当前系统中已创建的所有部门，每条记录包含以下信息：

| 字段 | 说明 |
|------|------|
| 部门名称 | 部门的显示名称 |
| 成员数量 | 该部门下的成员人数 |
| 主管人员 | 该部门的负责人 |
| 部门描述 | 部门的简要说明 |
| 状态 | 部门当前状态，通过开关切换启用 / 禁用 |
| 更新时间 | 部门信息的最后更新时间 |

![部门管理页面](/assets/组织架构/部门管理-页面基本信息-3.png)

---

## 新增部门

点击页面右上角的**新增部门**按钮，弹出新增部门弹窗。

填写以下信息后点击**确定**完成创建：

| 配置项 | 是否必填 | 说明 |
|--------|----------|------|
| 部门名称 | 必填 | 部门的显示名称 |
| 上级部门 | 选填 | 选择该部门所属的父级部门，不选则默认为顶级部门 |
| 主管人员 | 选填 | 选择该部门的负责人 |
| 部门描述 | 选填 | 填写部门的简要说明，最多 100 字 |
| 选择状态 | 必填 | 设置部门初始状态，默认为**启用**；选择**禁用**后该部门将停止使用 |

![新增部门](/assets/组织架构/部门管理-新增部门-1.png)

> **提示：** 部门创建后可在编辑页面随时修改名称、上级部门及启用状态。

---

## 添加子部门

在部门列表中，点击某条部门记录右侧操作栏中的**添加子部门**按钮，可在该部门下快速创建下级部门。

![添加子部门](/assets/组织架构/部门管理-操作-添加子部门-2.png)

弹窗中**上级部门**字段将自动填充为当前所选部门，填写子部门名称并设置状态后，点击**确定**完成创建。

> **提示：** 系统支持多层级部门结构，可根据实际组织架构灵活配置。

---

## 编辑部门

点击部门列表中某条记录右侧的**编辑**按钮，弹出编辑部门弹窗。

可修改以下信息：

| 配置项 | 说明 |
|--------|------|
| 部门名称 | 修改部门显示名称 |
| 上级部门 | 调整部门所属的父级部门 |
| 主管人员 | 修改该部门的负责人 |
| 部门描述 | 修改部门的简要说明 |
| 选择状态 | 启用或禁用该部门 |

修改完成后点击**确定**保存。

---

## 删除部门

点击部门列表中某条记录右侧的**删除**按钮，系统将弹出确认提示。

> **注意：** 删除操作不可恢复，且存在成员的部门无法直接删除，请先将成员迁移至其他部门后再执行删除。如需临时停用部门，建议使用**禁用**功能代替删除。
