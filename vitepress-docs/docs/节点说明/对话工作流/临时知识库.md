# 临时知识库节点

## 概述

临时知识库节点用于在对话工作流中动态接收用户上传的文件，并对其进行实时索引与检索。与私域知识库不同，临时知识库仅在当前对话会话中生效，不会持久化存储，也不会影响已有的历史文件索引。


## 核心特性

| 特性 | 说明 |
|------|------|
| 会话隔离 | 不受历史文件影响，每次初始化均为全新状态，不会重新索引旧文件 |
| 动态索引 | 用户在对话中上传文件后继续提问，系统会对新上传的文件自动触发索引 |
| 依赖私域知识库 | 必须先连接私域知识库，并在私域知识库中添加临时知识库后方可使用 |
| 输出限制 | 临时知识库的输出变量不可直接连接到模型节点，需通过知识库检索结果进行中转 |
| 图片解析 | 支持对文件中的图像内容进行解析，需配置图像处理模型及分析提示词 |



## 使用前提

1. 已创建并配置私域知识库
2. 在私域知识库设置中启用并添加临时知识库
3. 工作流中的临时知识库节点与私域知识库节点完成连接



## 节点配置说明

### 描述

可为当前节点添加备注描述，便于工作流维护与协作时理解节点用途。

### 输入变量

选择上游节点传入的文件变量，作为本次索引的文件来源。

**支持的文件类型：**

`txt`、`markdown`、`mdx`、`pdf`、`html`、`xlsx`、`xls`、`doc`、`docx`、`csv`、`eml`、`msg`、`pptx`、`xml`、`epub`、`ppt`、`md`、`htm`

> 单个文件大小限制：**< 100M**

### 文档解析服务

选择用于解析文档内容的服务引擎，默认为 **S-高级文本解析T**，适用于大多数文档格式，可提供更高质量的文本提取效果。

### 分段最大长度（Token）

设置文档在索引时每个分段的最大 Token 数，默认值为 **500**。

- 数值越小，检索粒度越细，适合精确问答场景
- 数值越大，上下文保留更完整，适合需要段落级理解的场景
- 可通过 `−` / `+` 按钮调整，或直接输入数值

### 图片解析

开启后，系统将对文件中包含的图像内容进行识别与分析，适用于含图表、截图等非纯文本文件。

| 配置项 | 说明 |
|--------|------|
| 文件图像处理 | 选择用于图像理解的模型，默认为 `gpt-4o-5`，可点击**设置**进行模型参数配置 |
| 图像分析提示词 | 输入引导模型分析图片的提示词，例如"请描述图中的数据表格内容" |

> 图片解析功能默认**开启**，若文件不含图像或无需图像理解，可关闭以节省处理资源。

### 初始化

点击**初始化**按钮，系统将对当前输入变量中的文件执行索引构建。初始化完成后，后续的知识库检索节点即可基于该文件内容进行检索。

> 每次对话开始时需重新初始化，上一次会话的索引不会保留。



## 输出变量

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `text` | string | 从上传文件中提取的文本内容 |

> 输出变量 `text` **不可直接连接模型节点**，需先经由知识库检索节点处理，再将检索结果传入模型节点使用。



## 典型使用场景

### 场景一：用户上传文档后即时问答

用户在对话中上传 PDF、Word 等文档，临时知识库对文件内容实时索引，后续提问可基于该文档内容进行检索与回答，无需提前入库。

### 场景二：会话级私有文件分析

适用于需要对用户当次上传的私有文件进行分析的场景，文件内容仅在本次会话中可用，会话结束后不保留，保障数据隐私。

### 场景三：含图表文件的智能解析

对于包含图片、图表或截图的文件（如 PPT、扫描版 PDF），开启图片解析功能后，系统可结合视觉模型对图像内容进行理解，实现图文混合内容的完整检索。



## 注意事项

- 临时知识库的输出变量**不可直接连接模型节点**，应先经由知识库检索节点处理后，再将检索结果传入模型节点
- 每次对话初始化时，临时知识库均为空，不会加载上一次会话的文件
- 若用户在同一对话中多次上传文件，每次上传后继续提问均会触发新的索引流程
- 开启图片解析时，请确保已正确配置图像处理模型，否则图像内容将无法被识别
- 文件大小需控制在 **100M 以内**，超出限制的文件将无法完成索引
